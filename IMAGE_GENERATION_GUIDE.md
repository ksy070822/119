# 결제 대란 RPG — 이미지 생성 가이드 (몰트봇 전달용)

> 이 문서는 몰트봇(Moltbot) 등 이미지 생성 AI에게 전달하여, 게임에 적합한 에셋을 생성하기 위한 사양서입니다.

---

## 1. 게임 개요 및 히스토리

### 1.1 게임 정체성
- **이름**: 결제 대란 RPG (Payment Crisis RPG)
- **장르**: 장애대응 의사결정 시뮬레이션 + RPG
- **테마**: "카카오 T 마을"을 배경으로 한 **결제 시스템 장애** 대응 스토리. 기술 장애를 판타지 RPG 메타포로 풀어냄.
- **핵심**: 정답/오답 없음. 모든 선택은 진행 가능하며, **리스크(조직 혼란, 대외 위험)** 만 변화. 플레이어는 판단하고, 동료의 조언을 받아 리스크를 완화함.

### 1.2 스토리 라인
1. **평화로운 카카오 T 마을** — 정상 운영 중
2. **심상치 않은 기운** — 결제 장애 징후
3. **결제 대란이 다가옴** — 위기 심화
4. **다섯 영웅 소집** — 각 역할(커뮤니케이터, 테크리더, 테크커뮤니케이터, 컨트롤타워, 리포터) 담당
5. **의사결정 여정** — 장애 인지 → 상황 전파 → 공지 판단 → 심화 공지 → 복구 → 후속 조치
6. **엔딩** — S/A/B/C 등급으로 해소 완료

### 1.3 시각 컨셉
- **메타포**: 실제 IT 조직·결제 장애를 **판타지 RPG**(영웅, 아이템, 보스)로 표현
- **톤**: JRPG 스타일의 따뜻한 색감 + 위기감을 주는 단계별 색 변화 (녹색→노랑→주황→빨강)
- **로케이션**: 카카오 T 마을(외부), 길드홀/컨트롤센터(실내)

## 2. 게임 구조 및 에셋 경로

### 2.1 폴더 구조

```
public/assets/
├── characters/          # 캐릭터 스프라이트
│   ├── communicator/
│   ├── tech_leader/
│   ├── tech_communicator/
│   ├── control_tower/
│   ├── reporter/
│   ├── boss/
│   └── monsters/
├── items/               # 캐릭터별 아이템
│   ├── communicator/
│   ├── tech_leader/
│   ├── tech_communicator/
│   ├── control_tower/
│   └── reporter/
├── maps/
│   ├── village/         # 외부 마을
│   └── guild/           # 실내 길드홀/컨트롤센터
└── ui/
    └── title.png        # 메인 타이틀 화면
```

### 2.2 씬(화면) 흐름
1. **타이틀** → 스토리 보기 or 캐릭터 선택
2. **인트로** → 6장면 시네마틱 (평화→위기→영웅 소집→타이틀)
3. **캐릭터 선택** → 5명 중 1명 선택
4. **마을** → 맵 이동, NPC와 대화, 아이템 획득
5. **컨트롤센터** → 보스(결제 대란) 시각화, 동료 조언
6. **엔딩** → S/A/B/C 등급

### 3.2 캐릭터별 이미지 규격

**필수 파일 (각 캐릭터 폴더):**

| 파일명 | 용도 | 권장 비율 | 비고 |
|--------|------|-----------|------|
| `idle.png` | 맵 위 서있기 (멈춤) | 세로형, 발 기준 하단 | 기본 포즈 |
| `walk_up.png` | 위로 이동 | 세로형 | 위쪽을 바라봄 |
| `walk_down.png` | 아래로 이동 | 세로형 | 정면 or 아래 (없으면 idle 폴백) |
| `walk_left.png` | 왼쪽 이동 | 세로형 | communicator, techLeader는 walk_left 없음 → idle 폴백 |
| `walk_right.png` | 오른쪽 이동 | 세로형 | 공통 필수 |
| `portrait.png` | 대화창 초상화 | 정사각형 권장 | 상반신 또는 얼굴 클로즈업 |
| `action.png` | 스킬/액션 포즈 | 세로형 | (선택) communicator, techLeader, reporter에 있음 |

**스타일 가이드:**
- **캐릭터**: JRPG/판타지 풍, 반쯤 리얼리스틱. 의상에 역할 컬러 포인트.
- **발 기준 앵커**: 이미지 하단이 발 위치 (맵 위에 서 있는 느낌).
- **일관성**: 5캐릭터 동일 비율·해상도 (예: 64×96px ~ 128×192px).

### 3.3 보스 (결제 대란)

| 파일명 | 용도 | 리스크 구간 |
|--------|------|-------------|
| `idle.png` | 기본 상태 | 0~25% |
| `angry.png` | 분노 상태 | 25~50% |
| `weakened.png` | 약화 상태 | 50~75% |
| `defeated.png` | 격파 상태 | 75~100% |

**의도**: "결제 대란"을 추상적/상징적 보스로 표현 (예: 붉은 에너지, 시스템 오류 비주얼, 악령 형태 등).

|--------|--------|

## 7. 이미지 생성 시 체크리스트 (몰트봇용)

### 7.1 캐릭터 생성 시
- [ ] 5캐릭터 모두 동일 비율·스타일
- [ ] 역할별 컬러(#FFD700, #4A90A4 등) 의상/악세서리에 반영
- [ ] portrait는 정사각형, 대화창용
- [ ] walk_* 시리즈는 발이 하단, 맵 위 서있는 구도

### 7.2 배경 생성 시
- [ ] village = 외부 마을, guild = 실내 컨트롤센터
- [ ] stage_1~5 (또는 1~4) 색감 일관: green→yellow→orange→red→recovering

### 7.3 아이템 생성 시
- [ ] base_* = 캐릭터 메인 도구, 아이콘 형태
- [ ] sub1~4 = 보조 아이템, 작은 아이콘

### 7.4 보스 생성 시
- [ ] idle → angry → weakened → defeated 순서로 위기/완화 시각화
